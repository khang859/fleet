How should editing work?

This is the core design fork. .env files are KEY=VALUE pairs — we can treat them as structured data, as raw text, or both. Click the one that feels right.

A · Structured editor

One row per variable. Inline edit, add/delete rows, auto-masked secrets with reveal, dup-key & format validation. Safest, most guided.

# Database DATABASE_URL=postgres://localhost:5432/app API_KEY=sk_live_3f9a PORT=3000

B · Raw text editor

Edit the file as text with syntax highlighting. Preserves comments, blank lines & ordering exactly. Familiar, but no guardrails or secret masking.

C · Hybrid recommended

Structured form by default (guardrails + masking), with a one-click toggle to raw text for comments/bulk edits. Best of both — more to build.

My lean: C (Hybrid) — the structured view gives you the click-feedback, validation, and secret-masking polish you asked for, while raw mode is the escape hatch for comments and edge cases that a pure form would mangle.
